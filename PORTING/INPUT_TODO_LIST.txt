================================================================================
1. KEYBOARD INPUT (input_sdl3.cpp)
================================================================================

  [100%] SDL_EVENT_KEY_DOWN / KEY_UP handling  — Processed in main loop
  [100%] SDL scancode → Atari KBCODE mapping  — Direct POKEY keyboard matrix path
  [100%] SDL scancode → ATInputCode mapping   — For input manager controller mapping
  [100%] ImGui keyboard capture check         — WantCaptureKeyboard guard
  [100%] Modifier keys (Shift, Ctrl, Alt)     — Mapped to Atari Shift/Control/Option
  [100%] Function keys (F1-F12)               — Mapped to emulator controls
  [100%] Arrow keys                           — Mapped per ATUIKeyboardOptions mode
  [100%] Special keys (Tab, Backspace, etc.)  — Full scancode table


================================================================================
2. GAMEPAD / JOYSTICK (joystick_sdl3.cpp)
================================================================================

  [100%] IATJoystickManager implementation    — ATJoystickManagerSDL3
  [100%] SDL_EVENT_GAMEPAD_* handling         — Button/axis events processed
  [100%] Gamepad button → Atari input mapping — XInput convention (A/B/X/Y/DPad)
  [100%] Analog axis mapping                  — Left stick → joystick directions
  [100%] Gamepad hotplug                      — SDL3 handles add/remove events
  [100%] Multiple gamepad support             — Tracked by SDL device ID
  [ N/A] Rumble / haptic feedback             — Not implemented in Windows Altirra
  [100%] Trigger axis as analog              — LT/RT mapped to JoyVert2/JoyVert4 with
                                               deadzone and power curve transforms


================================================================================
3. MOUSE INPUT
================================================================================

  [100%] SDL_EVENT_MOUSE_MOTION              — Processed in main loop
  [100%] SDL_EVENT_MOUSE_BUTTON_*            — Left/right/middle clicks
  [100%] ImGui mouse capture check           — WantCaptureMouse guard
  [100%] Relative mouse mode (paddle)        — SDL_SetWindowRelativeMouseMode via
                                               ATUICaptureMouse/ATUIReleaseMouse
  [100%] Mouse capture for paddle emulation  — ATUICaptureMouse toggles relative mode,
                                               auto-capture on left click, release on
                                               focus loss / ImGui capture / shutdown
  [100%] Absolute mouse mode (light pen)     — SDL_SetWindowMouseGrab for confining
                                               cursor to window
  [100%] Mouse → light pen position mapping  — Pixel to ANTIC beam coordinate
                                               transform via GetFrameScanArea(), plus
                                               pad position and virtual stick

                                               queries ATInputManager per port


================================================================================
5. TEXT INPUT
================================================================================

  [ N/A] SDL_EVENT_TEXT_INPUT handling         — Not needed for Atari keyboard.
                                                The Atari has a fixed physical keyboard
                                                matrix; SDL3 scancodes map directly to
                                                Atari keycodes (layout-independent).
                                                Windows uses WM_CHAR+character tables for
                                                the same purpose but SDL3's scancode path
                                                is the direct equivalent. ImGui handles
                                                its own text input for UI widgets.


================================================================================
6. FILE DROP
================================================================================

  [100%] SDL_EVENT_DROP_FILE handling          — Loads dropped file with VRWSafe + cold reset

All input types fully functional: keyboard (with function key toggle,
raw key release on focus loss), gamepad (with triggers, deadzone, power),
mouse (relative + absolute + beam + pad + virtual stick + buttons + wheel),
full input mapping editor, input setup, port assignment, quick map cycling.

Remaining architectural note: the virtual key map system (ATUIInitVirtualKeyMap)
is stubbed — keyboard layout customization and arrow key mode options take
effect through settings.cpp but not through the full VKeyMap dispatch.
The scancode path in input_sdl3.cpp handles the standard cases correctly.
